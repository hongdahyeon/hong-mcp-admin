# Frontend Overview

이 문서는 프로젝트의 프론트엔드 구조와 주요 결정 사항을 설명합니다.

## 1. .jsx vs .tsx 차이점

| 특징 | .jsx (JavaScript XML) | .tsx (TypeScript XML) |
| :--- | :--- | :--- |
| **언어** | JavaScript | TypeScript |
| **타입 체크** | 런타임에 확인 (동적 타입) | 컴파일 타임에 확인 (정적 타입) |
| **안정성** | 실행 전까지 오류 발견이 어려움 | 코드 작성 중 타입 오류 즉시 발견 |
| **IDE 지원** | 자동 완성 기능이 제한적임 | 강력한 자동 완성 및 리팩토링 지원 |
| **가독성** | 타입 코드가 없어 간결함 | 인터페이스 등 타입 명시로 의도 파악 용이 |

**결론:** 본 프로젝트는 코드의 안정성과 협업 효율을 높이기 위해 **TypeScript(.tsx)**를 기본으로 사용합니다.

## 2. 프론트엔드 구조 개편 (Walkthrough 요약)

최근 작업을 통해 다음과 같이 구조를 최적화했습니다.

### ⚙️ 설정 관련 변경
- **TypeScript 도입**: `tsconfig.json` 설정을 통해 프로젝트 전반에 타입 체크 적용
- **Vite 설정 최적화**: `vite.config.ts`에서 `@/` 별칭(Alias)을 설정하여 `src` 디렉토리 접근 용이성 확보
- **라우팅 도입**: `react-router-dom`을 사용하여 URL 기반 페이지 전환 처리
- **엔트리 수정**: `index.html`에서 호출하는 메인 파일을 `main.tsx`로 변경

### 📂 폴더 구조 및 역할
- `api/`: 서버와의 통신을 위한 Axios 인스턴스 및 API 호출 함수 관리
- `components/`: 버튼, 모달 등 재사용 가능한 UI 컴포넌트 (`common/`, `layout/`)
- `hooks/`: 비즈니스 로직을 분리한 커스텀 React Hooks
- `pages/`: 실제 URL 경로와 매칭되는 전체 페이지 컴포넌트 (View의 목적지)
- `routes/`: URL 경로와 컴포넌트 매핑 설정 (Navigation의 길잡이)
- `store/`: 전역 상태 관리 (예: 사용자 인증 정보, 테마 등)
- `types/`: 프로젝트 전역에서 사용되는 공통 타입 및 인터페이스 정의
- `utils/`: 날짜 포맷팅, 숫자 계산 등 순수 유틸리티 함수

### 💡 Routes vs Pages vs Components
1. **Routes**: **"어디로(URL)"**에 집중합니다. 특정 URL에 접근했을 때 어떤 페이지를 보여줄지 결정하는 '중앙 통제소'입니다.
2. **Pages**: **"무엇을(Page)"**에 집중합니다. 라우터에 의해 호출되며, 여러 컴포넌트를 조합하여 하나의 화면을 구성합니다.
3. **Components**: **"어떻게(UI/UX)"**에 집중합니다. 페이지를 구성하는 최소 단위(Atomic) 혹은 기능 단위의 조각들입니다.

## 3. 개발 가이드
- 새로운 컴포넌트 생성 시 반드시 `.tsx` 확장자를 사용하세요.
- 공통으로 사용되는 타입은 `src/types/`에 정의하여 공유하세요.
- 백엔드 주소는 `.env` 파일(추후 생성 예정)을 통해 관리할 것을 권장합니다.
